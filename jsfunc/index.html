Variables

Like many other programing languages, variables are a storage container that has a unique name. variables are declared using either the keyword var or let. This article from StackOverflow details when you would use var vs. let. For this demonstration, we are going to use var.
The following is a list of rules you must follow when naming variables: (1)

  • Names can contain letters, digits, underscores, and dollar signs.
  • Names must begin with a letter
  • Names can also begin with $ and _
  • Names are case sensitive (y and Y are different variables)
  • Reserved words (like JavaScript keywords) cannot be used as names

The following are examples of valid variable names: var cit261; var CIT_261; var _C_i_T;.
The following are examples of invalid variable names: var 261cit; (cannot begin with number) var Kitty_C@T; (can't use '@' symbol).
!! Don't forget that variable names are CaSe SeNsItIvE !!

Once we have created a variable, we can assign values to it. We use the Assignment Operator also known as the equal sign = .
We can assign different types for data to a variable. These include numbers (like 1, 256, 3.14 etc.), strings which are texts surrounded by quotes ("this is a string" or "100"), boolean (true or false), undefined is when a variable has been declared (i.e. var x;) but has not had a value assigned to it yet, null is an assigned value that means "no value", and object which will be talked about in another lesson. One neat thing about JavaScript is that it is an "untyped" language. That means that you don't have to tell it what type of information your variable is holding like you do in other languages such as Java or C++.
I could create a variable called "groupSize" and then assign it a value

var groupSize;
groupSize = 7;

Or I could do it all on one line.

var groupSize = 7;

Multiple variables can be declared and/or defined with a single statement.
var groupSize, groupNumber, groupName;
var groupSize = 7, groupNumber = 3, groupName = "Team 3";
Variables can also be assigned the value of another variable or an arithmetic value.
var x = 2; var y = x;
var x = 8 * 8;

Scope is somthing that needs to be kept in mind when working with variables. If a variable is declared inside a set of braces (curly brackets "{ }") it cannot be accessed outside of those braces, it has a local scope. If you want a variable to have a global scope, it needs to be declared outside of a function or set of braces. Basically, any variable can be accessed that is inside the codeblock (set of braces) you are working in, or above it.

Functions

Functions allow us to "do stuff" with javascript. We can make variables all day long, but without a function, they don't do us any good. Functions "are executed when "something" invokes them." (1) Another name for functions is methods. Functions allow us to write code that does something, and then reuse that same peice of code again and again, just by invoking it. Functions begin with the keyword function followed by the name of the function, function myFunction. After the name comes a set of parenthesis followed by the enclosing set of braces function myFunction () { }. The parenthesis are left empty if there are no parameters, or may contain 1 or more parameters. The body of the function contains all the instructions and is held between the beginning and ending braces. Sometimes functions return information to the entity that called or invoked it. This information is sent back using the return statement. As stated earlier, variables that are declared inside of a function have local scope, and can only be used inside that function. Once the function has finished, the variable gets deleted. If a variable is defined outside of a function, it has global scope. It can still be used inside the function, but when the function ends, the variable will persist. That global variable can be used and changed by different functions as well.
A function can be invoked by an event such as onclick. It can be embedded in the html with the <script> tags. It can even be invoked by another function. To invoke a function you write the name of the function followed by the parenthesis and put inside the parenthesis any arguments.
myFunction(x, y);
This code will call the function "myFunction" and will pass (send) two variables called 'x' and 'y' to the function. 'x' and 'y' will be used in the function as parameters. When something is being sent to a function, it is called and argument. When the argument gets to a function, it is called a parameter.

Parameters

When writing parameters into a function, the data type is not specified. The name is simply listed. If there is more than one parameter they are separated by a comma (param1, param2, param3). The name of a parameter does not have to match the name of the variable that is being passed as an argument. When the parameter is listed, it is basically declaring a new variable and then setting the value as the argument that was passed. If you have more than one parameter, how do you correctly match up the passed arguments so they don't get mixed up? They are assigned in the order they are recieved to the order they are listed. If I call a function myFunc(7, 3, "tree"); And my parameters are setup as function myFunc(plant, height, width), I will end up with plant = 7; height = 3; width = "tree", which doesn't really make much sense. I should have passed the "tree" argument first. If an aregument is missing, the corresponding parameter will be undefined. If I have a variable x that has a value of 10 and I sent it to the divideByTwo() function, what will x be when divideByTwo() finishes? The answer is 10. The value of x is not changed by a function. That is because areguments are Passed by Value, which means that a copy of the value of x is made and sent to the function. The original x does not get sent. When Objects are sent to a function, they are Passed by Reference, which means that anything done to the object in the function, will change the object.

Conditional Statements

Conditional statements allow for actions to be performed based on certain conditions. JavaScript has 4 conditional statements.

  1. if
  2. else
  3. else if
  4. switch

If, else, and else if are keywords followed by parenthesis. A statement that will evaluate to either true or false is placed inside the parenthesis. For an if statement, the action below is executed when the condition evaluated to true. For an else statement, the action is executed if the code is false. else and else if statements always follow an if statement. An action listed below an else if will execute if the if statement that came before the else if statement was false and the condition in the else if statement is true. Multiple if and else if statements can be used, but only 1 else statement.

It is possible to have multiple ifs and else ifs in one large conditional statement, but it is not the best way. When checking for a specific condition out of many, a switch statement can be the best option. A switch statement is setup as follows: 

        
        switch(expression) {
            case n:
                code block
                break;
            case n:
                code block
                break;
            default:
                code block
        }

"switch" is a keyword. Inside the parenthesis is placed the expressions to be evaluated. This could be a variable or any other expression. The 'n' is what we are comparing to the expression. If the expression evaluates to n, the the code block below is executed. After it is executed, the break statement exits the switch statement and it is finished. If no case evaluates to true, the default code block is executed.

Loops

Loops allow a code block to execute repeatedly until a certain condition is met. There are 4 types of loops in JavaScript:

  • for
  • for/in
  • while
  • do/while

                
                    for (var i = 0; i < 3; i++) {
                        code to be executed;
                    }

There are three statements separated by semicolons after the "for". The first is executed before the code block starts. The second is a conditional statement. If it evaluates to true then the code block is executed. After the code block is executed, the last statement (which increments i by 1) is executed. Then the loop starts over in the same order, except the first statement is not executed again.


The "for/in" loop is for looping through the properties of an object. We can look at this more after we learn about objects.
The while loop consists of a conditional statement followed by a code block. As long as the conditional statement evaluates to true, the code block will run repeatedly. Here is an example of a while loop: 

            
                while (1 > 0) {
                    Window.alert("1 is bigger than 0!");
                }

In this example as long as 1 is greater than 0, an alert will pop up on the screen.


If we are using a while loop and the condition were false the very first time, the code block would never run. Sometimes we want it to run once and then check a condition to see if it should run again. That is exactly what a do/while loop does. Here's an example: 

            
                do {
                    console.log("Hello");
                }
                while (1 < 0);

This code will run the first time, but then the condition evaluates to false and it will exit the loop.

Arrays

Arrays are a type of container variable that hold multiple values. It is declared the same as any other variable with one exception. I use var students to declare the variable name, but instead of setting it equal to a single value, I set it equal to square brackets with a comma separated list ["Adam", "Allen", "Dennis", "Irina", "Ricardo", "Scott", "Shanna Mae"]. Now my variable students contains 7 different values. Each value can be accessed with a key, which starts at '0' and goes up to the number of items in the array minus 1 (because we start at 0 instead of 1). If I want to access "Ricardo" in my students array, I would type students[4] because "Ricardo" is in the array slot 4 (counting from 0).

Associative Arrays

"Many programming languages support arrays with named indexes.
Arrays with named indexes are called associative arrays (or hashes).
JavaScript does not support arrays with named indexes.
In JavaScript, arrays always use numbered indexes."(1)

Arrays are actually a type of object, but they are a special type in that the keys used to access them are always numbers.
